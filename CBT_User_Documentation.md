# CBT Application Documentation for End Users

## Overview
The Computer-Based Testing (CBT) application is a comprehensive examination management system developed for BUK KANO. It enables secure online testing with multiple user roles, including students, instructors, administrators, and invigilators. The system ensures secure exam delivery, proper student authentication, and comprehensive exam management.

## User Roles and Responsibilities

### 1. Student User Experience

#### Accessing the System:
- Students access the system through the home page
- Login requires two credentials:
  - **Candidate Number**: Your unique identification number provided by your institution
  - **Ticket Number**: A secret code provided by the examination authority

#### Pre-Exam Process:
1. **Check-in Requirement**: Students must be physically checked in by an invigilator before accessing the exam
2. **Identity Verification**: The invigilator must verify your identity before granting access
3. **No Check-in Notice**: If you attempt to login without being checked in, you will be redirected to a "Not Checked In" page

#### Exam Interface:
1. **Navigation Panel**: 
   - Shows all questions with color-coded indicators
   - Unanswered questions appear in gray
   - Answered questions appear in green
   - Current question appears in blue

2. **Question Features**:
   - Multiple-choice questions with options A, B, C, D
   - Options are shuffled for security
   - Navigation between questions using "Next" and "Previous" buttons
   - Keyboard shortcuts (1-4 for options, arrow keys for navigation)

3. **Timer Display**:
   - Countdown timer shows remaining exam time
   - Shows any time extensions granted
   - Automatically submits exam when time runs out

4. **Answering Questions**:
   - Select an answer by clicking on the option
   - Answers are saved automatically to local storage
   - Can navigate freely between questions
   - Each question can be answered only once and saved

#### Submitting the Exam:
- Click the "Submit Exam" button when finished
- Confirm submission in the pop-up dialog
- After submission, you'll be redirected to a submission confirmation page
- Your answers are permanently recorded in the system

### 2. Instructor User Experience

#### Dashboard Overview:
- Access courses assigned by administrators
- View statistics: number of courses, active exams, total questions, and students
- Quick access to question banks and exam management

#### Course Management:
1. **View Assigned Courses**: Access all courses assigned by the administrator
2. **Add Questions**: Navigate to "Question Bank" to add questions to your course
3. **Create Exams**: Manage exams for your assigned courses
4. **Manage Students**: View and manage students enrolled in your courses
5. **View Results**: Access student performance and exam results

#### Question Bank Management:
- Add new questions with multiple options
- Edit existing questions
- Categorize questions by topic or type
- Preview questions before adding to exams

### 3. Administrator User Experience

#### Admin Dashboard:
- View system-wide statistics (total students, active courses, instructors)
- Monitor upcoming exams and recent submissions
- Manage user accounts and permissions

#### User Management:
- **Student Management**: Add, edit, or remove student accounts
- **Instructor Management**: Add, assign courses, and manage instructor accounts
- **Invigilator Management**: Assign invigilators to exams and sessions

#### Course and Exam Management:
- Create and manage courses
- Assign instructors to courses
- Create and activate exams
- Set exam parameters (duration, marks per question)
- Monitor exam status and results

#### Academic Session Management:
- Create academic sessions (semesters)
- Manage department assignments
- Control global session settings
- Handle ticket management for exams

### 4. Invigilator User Experience

#### Pre-Exam Setup:
- Access assigned exam and course information
- View exam duration, title, and activation date
- See list of students enrolled in the exam

#### Student Check-in Process:
1. **Student Verification**: 
   - Physical identity verification required
   - Check student photo against records
   - Verify student is on the enrollment list

2. **Check-in Action**:
   - Search students by name or candidate number
   - Filter by check-in status
   - Click "Check-in Student" button after verification
   - System confirms successful check-in with notification

3. **Student List Management**:
   - Real-time student status display
   - Visual indicators (gray for pending, green for checked-in)
   - View student photos by clicking on image icons
   - Filter students by status (all, pending, checked-in)

#### System Access:
- Students can only access exams after being checked in by an invigilator
- Invigilators must ensure all students are properly verified before check-in
- System tracks check-in times for accountability

## Getting Started Guide

### For Students:
1. Navigate to the CBT portal
2. Have your candidate number and ticket number ready
3. Wait for invigilator to physically verify your identity
4. Once checked in by the invigilator, login using your credentials
5. Read exam instructions carefully before starting
6. Answer questions and make sure to submit before time expires

### For Instructors:
1. Login with your instructor credentials
2. Access your assigned courses from the dashboard
3. Use the navigation sidebar to access question banks, exams, and students
4. Create or edit questions as needed
5. Monitor student progress and exam results

### For Administrators:
1. Login to the admin panel
2. Access student, instructor, and course management
3. Create academic sessions and assign courses
4. Monitor overall system performance
5. Handle user access and permissions

### For Invigilators:
1. Login using your invigilator account
2. View the assigned exam and course details
3. Verify the student list before exam begins
4. Physically verify each student's identity
5. Check students in using the invigilator panel

## Troubleshooting Common Issues

### Student Login Issues:
- If you can't login, ensure you've been checked in by the invigilator
- Verify your candidate number and ticket number are correct
- Contact your invigilator if you're sure you've been checked in

### Exam Issues:
- Answers are saved automatically, but ensure you submit before time expires
- If you lose internet connection, answers remain in local storage
- Refresh the page if the interface becomes unresponsive

### Technical Support:
- Use the support button on the home page
- Contact your institution's IT department
- Ensure you're using a compatible browser (Chrome, Firefox, Safari, Edge)
